# FnKey Landing Page — Project Overview

**Client:** Evgeny Oleynik (@evoleinik)
**Product:** FnKey (macOS voice-to-text app, Rust, open source)
**Deliverable:** Validation landing page + waitlist
**Status:** Research phase
**Timeline:** Starting March 17, 2026

---

## Problem Statement

FnKey has traction (107 upvotes on Product Hunt, #15 day rank) but:
- Visit Website link → GitHub README
- README = developer instructions (terminal, config, API keys)
- **Nontechnical users bounce at step 2**
- No marketing site, no brand presence outside GitHub

## Solution

Create a **validation landing page** that:
1. Explains value prop in human language (not CLI)
2. Shows demo (FnKey UI, privacy features)
3. Captures email for early adopters + future updates
4. Links to GitHub for developers
5. Maintains open source positioning (no upsell)

## Success Metrics

- ✓ Higher conversion from PH → mailing list
- ✓ Improved GitHub discoverability
- ✓ Clarified positioning vs competitors
- ✓ Nontechnical audience entry point
