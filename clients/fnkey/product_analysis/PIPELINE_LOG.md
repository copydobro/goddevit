## ФАЗА 1: Позиционирование и ICP
**Статус:** ✅ Завершена
**Документ:** `02_POSITIONING_ANALYSIS.md`
**Gaps найдено:** 7

### Что сделали
Прогнали данные FnKey через 4 скила последовательно.

### Скилы и порядок

| Скил | Задача | Порядок |
|------|--------|---------|
| `problem-definition` | Найти реальную боль, не описание фичи | 1 — база для всего остального |
| `value-architecture` | 3 слоя ценности: функция / статус / когниция | 2 — понять ценность до позиционирования |
| `positioning-messaging` | ICP, конкурентная рамка, messaging hierarchy | 3 — строится на боли и ценности |
| `neuro-foraging-cjm` | Путь пользователя, точки отвала, Travel Costs | 4 — показывает где ценность теряется |

### Ключевые выводы
- **Реальная боль:** разрыв между скоростью мышления и скоростью передачи контекста AI. Evgeny сам это формулирует в dev.to: "3x longer prompts = 3x better output". В README этого нет.
- **Слой ценности:** FnKey управляет только функциональным слоем. Якорь "$0 vs $240/год Wispr Flow" нигде не выставлен. Статусный слой есть (Evgeny = Antler CTO), но один носитель не создаёт паттерн.
- **Точка отвала:** PH → README → первый экран с terminal commands. BRR высокий (конкуренты в 2 клика), FRR падает сразу. Economic users уходят здесь.

## ФАЗА 6 (внеплановая): ICP-ревизия и market research
**Статус:** ✅ Завершена
**Документы:** `ICP_ANALYSIS.md`, `10_COPY_V2.md`
**Причина:** в процессе написания отчёта обнаружили что ICP был заявлен как факт на основе одного dev.to поста. Это методологическая ошибка — создала внутренние противоречия (ICP = технари, но рекомендации для нетехнических).

### Что сделали

**1. Создали ICP-методологию** (`icp-definition` skill)
Два режима: Mode A (post-launch signal mining) и Mode B (pre-launch hypothesis building).
Ключевые добавки vs первой версии:
- Struggling moment — операциональное определение: ситуация → обходной путь → последствие
- Фаундер ≠ ICP сценарий (Сценарий B)
- Приоритизация гипотез без данных: острота × доступность канала × близость фаундера
- Валидация без интервью: тест в 2-3 каналах одновременно разделяет ICP-ошибку от канальной

**2. Прошли Mode A для FnKey**
- Stargazers: 24/44 профиля → подтверждены "разработчики", не подтверждены "Claude Code users" специфически
- PH comments: недоступны (403)
- GitHub issues: пусто (0 issues)
- Вывод: нужны внешние данные

**3. Провели market research**
Промпт по методологии icp-definition → Perplexity → 35+ цитат с 6 платформ.
Файл: `revenue_and_growth/01_market_research/compass_artifact_wf-9151bd32...md`

Ключевые находки:
- ICP подтверждён на ВЫСОКОМ уровне уверенности
- Второй слой боли обнаружен: не только скорость, но self-censorship при наборе
- Wispr Flow "extremely buggy" — это главный аргумент для FnKey, не только цена
- Claude Code /voice (март 2026) — новый конкурент, примитивный, с lag
- 6 самодельных voice tools на Show HN за 2025–2026 — поведенческое доказательство pain

**4. Обновили документы**
- `ICP_ANALYSIS.md` — полный ICP-документ, уверенность ВЫСОКАЯ
- `02_POSITIONING_ANALYSIS.md` — убраны утверждения про нетехническую аудиторию
- `05_GTM_GAPS.md` — Gap 2 (TTFV) и Gap 9 (конкурентная рамка) переформулированы
- `06_REPORT.md` — ICP секция обновлена, конкурентная картина дополнена
- `07_30DAY_PLAN.md` — README hero обновлён до V2 copy, добавлены social proof assets
- `10_COPY_V2.md` — полный копирайт V2 для README и 5 экранов лендинга

### Что изменилось в рекомендациях

| Было | Стало | Причина |
|------|-------|---------|
| "Нетехническая аудитория уходит" | Убрано | Противоречие с ICP = технари |
| Gap 9: цена vs Wispr Flow | Gap 9: цена + баги + Claude /voice lag | Market research |
| README hero: generic speed | README hero: AI coding context + filtering | ICP + реальный язык |
| Social proof: нет | Social proof: Karpathy, Allie Miller, OpenAI exec | Research дал цитаты |
| Конкуренты: Wispr Flow, SuperWhisper | + Claude Code /voice (март 2026) | Новый entrant |

### Урок для следующего кейса
ICP нельзя выводить из одного источника. Даже если это сам фаундер.
Методология: сначала определить режим (A или B), затем собрать минимум 20 цитат,
затем проверить на противоречия с продуктовыми рекомендациями.
