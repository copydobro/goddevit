## Резюме

FnKey — macOS-инструмент голосового ввода на Rust. Зажми Fn, скажи, вставь — стриминг в реальном времени через Deepgram Nova-3, без задержек, GPL-3.0, бесплатно.

**Продукт работает. GTM — нет.**

107 апвоутов на Product Hunt с аккаунта с 35 фолловерами в Twitter. 43 звезды на GitHub. Листинг в Rust-комьюнити. Два поста на dev.to. Потом тишина.

У FnKey есть всё что нужно developer-инструменту для органического роста — реальный use case, технический фаундер с репутацией, бесплатная модель которая сносит конкурентов по цене. Чего не хватает — упаковочного слоя который конвертирует интерес в установку, и системы дистрибуции которая не зависит от единственного запускового события.

Аудит нашёл **15 конкретных, устранимых gap-ов**. Восемь из них критические.

## Находка по ICP

**Заявленный ICP (подразумевается):** все macOS-пользователи которые хотят голосовой ввод.

**Реальный ICP (подтверждён market research, уровень уверенности: ВЫСОКИЙ):**
Solo developer / indie hacker на macOS, ежедневно использующий Claude Code или Cursor как основной рабочий инструмент. Отправляет 20–70 промптов в день. Уже пробовал Wispr Flow или SuperWhisper — недоволен ценой или качеством.

Три уровня доказательств (35+ цитат, 6 платформ, март 2026):
- **Прямые цитаты:** "The bottleneck is our keyboard" (dev.to), "The major bottleneck is how fast I can express my ideas" (Cursor Forum), "typing was just too slow" (HN)
- **Поведение:** 6+ самодельных voice tools запущено на Show HN за 2025–2026. Karpathy, Andrew Chen, Allie Miller публично перешли на voice workflow.
- **Рынок:** OpenAI Head of Codex Product назвал human typing speed "the underappreciated limiting factor to AGI". Anthropic встроила /voice в Claude Code (март 2026).

**Второй слой боли** (из research, глубже скорости): разработчики не просто медленно печатают — они **фильтруют мысли** пока печатают. "A prompt that would have been 3 sentences typed became 2 paragraphs spoken." Claude получает урезанную версию контекста — отсюда лишние итерации.

**Обновление конкурентной картины:**
- Wispr Flow: главный конкурент, но "extremely buggy, cuts out randomly, worst customer service" — пользователи остаются только потому что voice input слишком важен
- Claude Code /voice (март 2026): новый встроенный конкурент, но 500ms lag, пропускает начало фраз, только транскрипция
- FnKey окно: бесплатный + streaming (не batch) + privacy (mic hold-only) + open source

Этот разрыв между реальным ICP и упаковкой продукта — корневая причина большинства gap-ов ниже.

## 15 GTM Gap-ов

> Формат каждого gap-а: Источник → Проблема → Действие → Ожидаемый эффект

### КРИТИЧЕСКИЕ — исправить первыми

**Gap 1 — README начинается с Installation, а не с ценности**
- Фаза: Упаковка
- Источник: прочитали README напрямую. Первая строка после заголовка и скриншота — `brew tap copydobro/fnkey`. Нет ни одной строки объясняющей зачем это нужно.
- Проблема: Посетитель GitHub открывает README и сразу видит команду установки. Нет постановки проблемы, нет value prop, нет причины читать дальше. Человек который не знает продукт — уходит.
- Действие: Добавить 2–3 строки перед Installation: struggling moment + "скажи всё что у тебя в голове — Claude получает полный контекст, не отфильтрованную версию"
- Ожидаемый эффект: снижение отказов на GitHub, рост конверсии visitor→install

**Gap 2 — TTFV = 9 шагов. Groq не предложен как quick start. Нет onboarding wizard.**
- Фаза: Упаковка / Онбординг
- Источник: прошли через README Installation секцию пошагово и посчитали реальные действия пользователя.
- Текущие 9 шагов:
  1. `brew tap copydobro/fnkey`
  2. `brew install fnkey`
  3. Открыть браузер → перейти на deepgram.com
  4. Создать аккаунт Deepgram
  5. Найти раздел API Keys в интерфейсе
  6. Создать ключ
  7. Скопировать ключ
  8. Открыть терминал → редактировать `~/.zshrc`
  9. Добавить `export DEEPGRAM_API_KEY=...` → `source ~/.zshrc`
- Проблема 1: Groq поддерживается (это в README), но не предложен как точка входа — хотя он бесплатный и не требует аккаунта с картой. Deepgram стоит по умолчанию.
- Проблема 2: Шаги 8–9 требуют редактирования shell-конфига вручную. Для нетехнического пользователя это терминальный барьер. Для технического — просто раздражение.
- Действие A: Переструктурировать README — "Quick Start (Groq, бесплатно)" как шаг 1, Deepgram как "Production upgrade".
- Действие B: Добавить onboarding wizard при первом запуске приложения — окно Setup с выбором провайдера, кнопкой "Get API Key" (открывает нужную страницу) и полем для вставки ключа прямо в UI. Приложение сохраняет ключ само. Шаги 8–9 исчезают полностью.
- С wizard: 9 шагов → 6. Устраняется всё что связано с ручной правкой конфига.
- Ожидаемый эффект: ниже drop-off на шаге с API-ключом, продукт становится доступным для нетехнической аудитории

**Gap 3 — Нет сайта. PH "Visit Website" → README на GitHub**
- Фаза: Упаковка
- Источник: проверили ссылку "Visit Website" на PH-странице FnKey — ведёт на github.com/copydobro/fnkey. Концепт лендинга существует по адресу copydobro.github.io/fnkeysite/ но не подключён к PH.
- Проблема: PH-аудитория неоднородна — среди 107 апвоутов есть не только разработчики. Нетехнический посетитель попадает в страницу с `brew tap` и уходит. Даже технический посетитель ожидает маркетинговый сайт, а не README.
- Действие: Задеплоить существующий концепт лендинга и обновить ссылку на PH
- Ожидаемый эффект: снижение Travel Cost с PH, рост конверсии PH→install

**Gap 4 — Core value prop существует, но скрыт в одном посте**
- Фаза: Позиционирование
- Источник: нашли формулировку "voice gives 3x longer prompts = 3x better AI output" в dev.to посте "Voice Transcription is a Superpower for Developers" (evoleinik, 2025). Проверили README и PH-листинг — этой формулировки там нет.
- Проблема: Лучшее объяснение ценности продукта живёт не там где его найдут новые пользователи. GitHub README и PH listing — основные точки входа. dev.to пост — нет.
- Действие: Перенести core insight в README hero: "скажи всё что у тебя в голове — Claude получает полный контекст, не отфильтрованную версию"
- Ожидаемый эффект: выше понимание продукта при первом визите, выше star rate

**Gap 5 — Тайтл на PH описывает реализацию, а не результат**
- Фаза: Позиционирование
- Источник: прочитали текущий тэглайн на PH-странице FnKey — содержит "with Deepgram stream" как ключевой дескриптор.
- Проблема: "Deepgram stream" — деталь реализации. Посетитель который не знает Deepgram не получает ни одного сигнала о том что продукт делает и для кого. Тэглайн должен отвечать на вопрос "зачем мне это" — сейчас отвечает на вопрос "как это работает".
- Действие: Обновить тэглайн: "FnKey – voice input for Claude Code and Cursor, zero delay, free"
- Ожидаемый эффект: выше CTR из поиска на PH, лучшее первое впечатление у незнакомой аудитории

**Gap 6 — Show HN не опубликован**
- Фаза: Дистрибуция
- Источник: поиск по Hacker News (news.ycombinator.com) по запросам "FnKey" и "copydobro" — результатов нет. Проверили профиль автора на HN — публикаций нет.
- Проблема: Hacker News Show HN — исторически самый высокоimpact органический канал для OSS developer tools на Rust. Инструменты сопоставимого класса (например, Zed, fish shell, Helix) получали 200–500 звёзд с одного Show HN поста. FnKey идеально подходит под аудиторию HN: Rust, macOS, AI coding workflow — этот канал не использован вообще.
- Действие: Опубликовать "Show HN: I built a voice-to-text tool for Claude Code in Rust — Hold Fn, speak, paste"
- Ожидаемый эффект: +100–200 звёзд в течение 24 часов, новый пул аудитории отличный от PH

**Gap 7 — После запуска на PH — полная тишина**
- Фаза: Post-launch
- Источник: проверили все каналы автора после даты PH-запуска. dev.to — оба поста датированы до запуска. X/Twitter (35 фолловеров) — нет постов про FnKey после PH. LinkedIn — нет упоминаний результатов запуска.
- Проблема: 107 апвоутов — это аномалия для аккаунта с 35 фолловерами. Это значит что продукт попал в чужие сети. Эти люди готовы были поддержать — но никто не попросил их написать отзыв, поделиться опытом, или вернуться. Момент социального доказательства упущен.
- Действие: Написать пост "Что произошло после 107 апвоутов на PH" + запросить 3–5 testimonials у авторов позитивных PH-комментариев
- Ожидаемый эффект: social proof для следующего запуска, возвратный трафик, материал для дистрибуции

**Gap 8 — Нет owned канала. Каждый запуск начинается с нуля.**
- Фаза: Дистрибуция
- Источник: проверили репозиторий — GitHub Discussions отключены. README не содержит ни одной ссылки на email, newsletter, Discord или любой другой способ подписаться на обновления.
- Проблема: После каждого запуска аудитория рассеивается. Нет механизма который конвертирует разового посетителя в подписчика. Следующий запуск начинается с той же базы — 35 фолловеров в X.
- Действие: Включить GitHub Discussions + добавить в README призыв "Star to follow updates"
- Ожидаемый эффект: каждый следующий запуск стартует теплее, накапливается база

### СРЕДНИЕ — высокий импакт, требует 1–3 часов

**Gap 9 — Конкурентная рамка отсутствует полностью**
- Фаза: Позиционирование
- Источник: конкурентный анализ + market research (цитаты пользователей Wispr Flow)
- Проблема: ICP уже использует Wispr Flow или SuperWhisper и недоволен. FnKey нигде не объясняет почему он существует как альтернатива. Три аргумента не коммуницированы ни в одной точке входа: цена ($0 vs $240), надёжность (Wispr Flow "cuts out randomly"), скорость (Claude Code /voice имеет 500ms lag).
- Действие: Добавить в README после hero competitive block:
  "Free forever. Wispr Flow costs $240/year and cuts out mid-sentence. Claude Code /voice has 500ms lag. FnKey: streaming, open source, mic active only while you hold the key."
- Ожидаемый эффект: закрывает сравнение для тех кто уже пробовал конкурентов — а это большинство ICP

**Gap 10 — GIF демонстрирует механику, а не use case**
- Фаза: Демо
- Источник: README содержит GIF (demo.gif). Судя по контексту README — демо показывает работу инструмента в общем виде, без конкретного приложения на фоне.
- Проблема: Основной use case — голосовой ввод в Claude Code или Cursor. Если GIF показывает текст появляющийся в пустом поле — непонятно зачем это нужно. Если показывает промпт появляющийся в Claude Code — всё очевидно без слов.
- Действие: Переснять GIF: Hold Fn → произносится Claude-промпт → текст появляется в поле ввода Claude Code в реальном времени
- Ожидаемый эффект: GIF продаёт use case, а не абстрактную механику

**Gap 11 — Нет badges в README**
- Фаза: Social Proof
- Источник: прочитали README — над Installation нет ни одного badge.
- Проблема: Badges (звёзды, лицензия, последний релиз) — стандартный trust signal для OSS. Их отсутствие не критично, но проект выглядит менее зрелым по сравнению с аналогами. Установить за 10 минут.
- Действие: Добавить shields.io badges под заголовок: GitHub stars, license, latest release
- Ожидаемый эффект: визуальный сигнал активности и доверия выше fold

**Gap 12 — Reddit не задействован**
- Фаза: Дистрибуция
- Источник: поиск по Reddit по запросам "FnKey voice macOS", "copydobro fnkey" — результатов нет ни в r/macapps, ни в r/rust, ни в r/ClaudeAI.
- Проблема: Три комьюнити с разной мотивацией — macOS-пользователи, Rust-разработчики, пользователи Claude — не охвачены. Каждое из них является естественной аудиторией для FnKey по разным причинам.
- Действие: Три отдельных поста с разными углами, с интервалом 2 дня
- Ожидаемый эффект: органический трафик из трёх комьюнити

**Gap 13 — Нет PH Upcoming страницы для следующего запуска**
- Фаза: Подготовка к PH
- Источник: проверили Product Hunt — страницы "Upcoming" для FnKey нет.
- Проблема: Первый запуск прошёл без предварительного прогрева аудитории — об этом говорит диспропорция 107 апвоутов при 35 фолловерах (голоса пришли органически, а не от подготовленной базы). Следующий запуск без Coming Soon повторит ту же ситуацию.
- Действие: Создать PH Upcoming прямо сейчас, начать собирать фолловеров
- Ожидаемый эффект: выше velocity в первый час следующего запуска → выше шанс попасть в featured

FnKey — это не проблема дистрибуции. Это проблема упаковки и последовательности. У продукта есть доказательство спроса (107 апвоутов от аккаунта с 35 фолловерами — это аномалия, что-то в продукте резонирует). Задача — сделать этот резонанс видимым в каждой точке входа.
