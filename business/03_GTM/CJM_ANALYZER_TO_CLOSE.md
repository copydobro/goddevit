# CJM: README Analyzer → Оплата → Онбординг
> Фреймворк: Нейро-Форажинг (MVT)
> Старт: пользователь попал на README Analyzer
> Финал: клиент в Notion-воркспейсе, бриф заполнен, работа идёт

---

## Карта пути: 7 патчей

```
[1] INTRO        →  [2] INPUT       →  [3] ANALYZING   →  [4] RESULT
                                                               ↓
[7] NOTION WS    ←  [6] PAYMENT     ←  [5] OFFER TOGGLE
```

---

## Патч 1 — INTRO screen

**Что происходит:** Пользователь читает анимированный текст. Видит sample findings. Принимает решение: вставить README или закрыть.

**FRR:** Высокий — текст конкретный, sample output показывает формат результата, любопытство активировано.
**BRR:** Низкий — бесплатных README-анализаторов с такой конкретикой нет на рынке.
**Travel Costs:** Минимальные — нужно только нажать ENTER или кликнуть.

**Information Scent:** `[ FAIL ] / [ WARN ]` формат — визуально обещает "ты получишь такое же для своего README".

**Риск:** Пользователь не дожидается конца анимации и уходит.
**Решение:** Typewriter speed откалиброван на быстрый темп. Клик в любом месте переходит дальше.

---

## Патч 2 — INPUT screen

**Что происходит:** Пользователь вставляет README или загружает файл. Единственное действие.

**FRR:** Зависит от скорости вставки — это его данные, барьер минимален.
**BRR:** Нет конкурирующих действий на этом экране.
**Travel Costs:** ⚠️ Когнитивный холм — "а вдруг это сохранит мой README где-то?" Нет явного упоминания что данные не сохраняются.

**Действие:** Добавить под textarea одну строку:
```
Processed locally. Nothing stored.
```
**Механизм:** Снижает dACC сигнал подозрения → убирает паузу перед вставкой.

**Information Scent:** Кнопка "ANALYZE README" — явный призыв. Drag-and-drop зона — альтернативный путь для тех кто не хочет копировать.

---

## Патч 3 — ANALYZING screen

**Что происходит:** Анимация `[ .. ]` пока LLM работает. Пользователь ждёт.

**FRR:** ⚠️ Временно нулевой — ожидание без ценности.
**Travel Costs:** Каждая секунда ожидания = нарастание сигнала в dACC.

**Что уже сделано правильно:** Анимация с блинкингом заменяет пустой экран — даёт ощущение работы.

**Что можно добавить:** Одна строка во время анализа с прогрессом:
```
> Checking problem statement...
> Checking value proposition...
> Checking demo materials...
```
**Механизм:** Progress = Information Scent = мозг видит движение к ресурсу = терпение выше.

---

## Патч 4 — RESULT screen

**Что происходит:** Пользователь видит свои findings с `[ OK ] / [ WARN ] / [ FAIL ]`, score, tips на каждый gap.

**FRR:** 🔥 Максимальный — первый момент реальной ценности. Персонализированный результат.
**Нейробиология:** Вентральный стриатум активирован — дофаминовый отклик на "это про меня".

**Ключевой принцип:** Cost of Inaction показывается ЗДЕСЬ — когда gap уже найден и боль конкретна.

**Структура каждого FAIL/WARN:**
```
[ FAIL ] No problem statement in first 5 lines   (0/6)
→ Fix: Add 2 sentences above the fold describing the pain you solve.
→ Cost: TOAST UI rewrote only their README — conversion 4% → 20%.
   At your traffic level this gap costs ~4 months of organic growth per month.
```

**Information Scent к офферу:** Под findings — переход к предложению должен быть логичным продолжением, не прерыванием.

**Pre-CTA текст (мост):**
```
These gaps are symptoms. The GTM strategy underneath is the diagnosis.
```
Не "хочешь мы это починим?" — это выпрашивание. А наблюдение которое подводит к следующему шагу.

---

## Патч 5 — OFFER TOGGLE

**Что происходит:** Пользователь разворачивает блок с описанием аудита. Читает что входит, сколько стоит.

**FRR:** Должен быть высоким — каждая строчка оффера должна давать микро-дозу ценности.
**Travel Costs:** ⚠️ Длинный текст = когнитивная нагрузка. Economic-тип уйдёт не дочитав.

**Решение:** Структура оффера под Economic-тип:
```
=== THE COLD START AUDIT ===

→ Traction Gap Map      ranked gaps, specific fixes, sources cited
→ 30-Day Protocol       what to do first, second, third
→ 5-Gap Guarantee       we find 5+ gaps or full refund

$500. Async. 7 days.

[BOOK YOUR AUDIT → $500]
```
Всё важное — в первых 5 строках. Дальше — детали для Exhaustive-типа.

**dACC снижение:** Гарантия убирает главный сигнал риска — "а вдруг это будет пустышка?"

---

## Патч 6 — PAYMENT

**Что происходит:** Пользователь кликает CTA → попадает на страницу оплаты.

**Нейробиология:** Этап покупки — конфликт префронтальной коры (Travel Costs, риск) и миндалевидного тела (доверие, желание результата).

**Текущий BRR:** Любой конкурент с более простым процессом оплаты = выше шанс ухода.

**Travel Costs которые убивают здесь:**
- Обязательная регистрация перед оплатой → ❌ убрать
- Непонятно что будет после оплаты → ❌ написать явно
- Форма с 10 полями → ❌ минимум полей

**Что показать на payment странице:**
```
After payment you'll receive a link to your personal Notion workspace.
Everything happens there: onboarding, brief, deliverables, updates.
No calls required.
```
**Механизм:** Снижает неопределённость → снижает Travel Costs → выше конверсия оплаты.

**Инструмент:** Stripe Checkout (минимальная форма) + автоматический webhook → создание Notion воркспейса.

---

## Патч 7 — NOTION WORKSPACE (онбординг)

**Что происходит:** Сразу после оплаты клиент получает ссылку на персональный Notion-воркспейс.

**Нейробиология:** ERN-купирование — немедленное подкрепление правильности решения. Мозг должен получить подтверждение "я не ошибся" в течение 60 секунд после оплаты.

**Структура воркспейса:**

```
📋 [Client Name] — GTM Strategy Audit
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

STATUS: 🟡 Brief pending

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
STEP 1 — Brief (10 min)            ← клиент заполняет
STEP 2 — Contract (2 min)          ← подписывает онлайн
STEP 3 — Results                   ← мы добавляем сюда
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[ Brief: 3 вопроса ]
[ Contract: одностраничный ]
[ What to expect: примеры результатов ]
[ Contact: @handle для быстрой связи ]

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
DELIVERABLES (добавляем сюда)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
→ Traction Gap Map          🔴 In progress
→ 30-Day Protocol           ⬜ Pending
```

**Почему это лучше чем "давайте созвонимся":**

| Традиционный путь | Notion-first путь |
|---|---|
| Оплатил → жди письма | Оплатил → сразу в воркспейсе |
| Созвон для онбординга | Асинхронно, в своё время |
| "Где мои результаты?" | Всё в одном месте |
| Статус непонятен | STATUS виден всегда |
| Новый документ каждый раз | Один воркспейс, всё накапливается |

**FRR после оплаты:** Максимальный — клиент сразу видит структуру, понимает что делать, ERN не формируется.

**Travel Costs на онбординге:** Минимальные — 3 вопроса в брифе, одностраничный договор, никаких звонков.

## Автоматизация: Stripe → Notion

**Технический путь:**
```
Клиент платит (Stripe Checkout)
    ↓
Stripe Webhook → Zapier/Make
    ↓
Дублировать Notion template → новая страница с именем клиента
    ↓
Отправить ссылку клиенту (Stripe confirmation email кастомизировать)
    ↓
Уведомление нам (Telegram/email) — новый клиент, бриф ждёт
```

**Notion template нужно создать один раз.** Каждый новый клиент = копия шаблона с его именем.
