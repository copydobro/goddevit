## 1. Поиск проектов (Inbound по концептам)

### Где искать продукты без сайта

| Источник | Фильтр | Сигнал |
|---|---|---|
| GitHub | ≥20 stars, нет website link в профиле репо | Популярен но не упакован |
| Product Hunt | Запущен ≥30 дней, upvotes ≥50, нет сайта или сайт = GitHub | Есть аудитория, нет конверсии |
| HN Show HN | Комментарии "нет нормального сайта" или "хочу но не умею" | Сам фаундер признаёт проблему |
| r/SideProject | Посты "запустил проект, как упаковать?" | Тёплый лид — сам ищет решение |

### Критерии выбора проекта для концепта

- Продукт понятен за 30 секунд (можно объяснить одним предложением)
- Есть реальные пользователи (stars, upvotes, комментарии)
- Фаундер активен (недавние коммиты, PH-активность, соцсети)
- Нет конкурирующего агентства уже работающего с ним
- Продукт интересен нам (мы понимаем аудиторию)

## 2. OSS Contributor Path

Альтернативный вход — не через питч, а через вклад.

**Последовательность:**
1. Fork репозитория
2. Улучшить README: problem statement, GIF, badges, description ≤12 слов
3. Улучшить маркетинговые тексты (GitHub description, topics)
4. Открыть PR с объяснением: "улучшил позиционирование, вот почему"
5. В PR или отдельно: "также сделал концепт лендинга — вот ссылка"
6. Фаундер видит качество работы до любого разговора о деньгах

**Почему работает:**
- OSS фаундеры ценят контрибьюторов
- PR — это доказательство компетенции, не питч
- Фаундер сам решает смотреть или нет — нет давления

## 3. Outreach после концепта

**Формула сообщения (проверена на FnKey):**
```
Наткнулся на [продукт] — [одна конкретная наблюдение о проблеме].
Сделал концепт: [ссылка]
Если зайдёт — готов доделать, мне нужен кейс в портфолио.
```

**Каналы (в порядке приоритета):**
1. LinkedIn (если фаундер там активен)
2. Twitter/X @упоминание
3. Email (через сайт, GitHub profile, илиevoleinik.com-style)
4. GitHub Discussion / Issue (как последний вариант)

**Follow-up если нет ответа:**
- День 1: первое сообщение
- День 5–7: один follow-up, другой канал или другой угол
- День 14: финальный touch ("просто хотел убедиться что дошло")
- После 14 дней: не преследовать, двигаться дальше

## 4. Контент-стратегия: кейсы → соцсети

Каждый завершённый концепт = контент на 2–3 недели.

### 10 форматов постов из одного кейса

| # | Формат | Угол | Платформа |
|---|---|---|---|
| 1 | Teardown | "Нашёл продукт с X звёздами и без сайта. Вот что он теряет" | X, LinkedIn |
| 2 | Before/After | Скрин до (README) vs после (концепт) | X, LinkedIn |
| 3 | Insight из исследования | "Проанализировал 8 конкурентов. Неожиданный вывод: [инсайт]" | X |
| 4 | Positioning decision | "Почему взяли янтарный а не фиолетовый" | X, LinkedIn |
| 5 | Copy breakdown | "Заголовок который мы написали и почему именно он" | X |
| 6 | Privacy/OSS angle | "Единственный инструмент где 'нет сервера' — не маркетинг а архитектура" | LinkedIn |
| 7 | Numbers | "Show HN с правильным README: +180 stars. С неправильным: +12" | X, LinkedIn |
| 8 | Process post | "Как мы за 3 дня собрали бренд-стратегию для OSS продукта" | LinkedIn |
| 9 | Builder story | "Фаундер строил это для себя. Вот как мы это упаковали" | X, LinkedIn |
| 10 | Meta / результат | "Отправили концепт. Вот что произошло дальше" | X, LinkedIn |

### Длинная форма (одна статья на кейс)

**Площадки:** vc.ru, DEV.to, Hashnode, Medium
**Структура статьи:**
1. Проблема: почему хороший продукт остаётся невидимым
2. Исследование: что мы изучили (конкуренты, аудитория, позиционирование)
3. Решение: ключевые дизайн и copy решения + почему именно они
4. Результат: что получилось + ссылка на концепт
5. Открытый финал: что дальше (предложение фаундеру + потенциальным клиентам)

**Эта статья одновременно:**
- Кейс-стади для портфолио
- Лид-магнит для похожих фаундеров
- SEO-контент ("landing page for OSS project")
- Social proof для следующего питча

## 5. Лид-магнит

**"GitHub + PH Traction Scorecard"** — публичная версия нашего github-traction-audit скилла.

Формат: одностраничный PDF / интерактивный чеклист
- 15 вопросов о репо и PH-странице
- Итог: "твой продукт теряет X% потенциального трафика из-за Y"
- CTA: "хочешь полный аудит — оставь ссылку"

Распространение:
- Ссылка в каждом teardown-посте
- В шапке LinkedIn профиля
- В GitHub profile README
- В Discord как welcome-message

## 6. Community (Discord)

Минимальная структура:
```
#welcome       → автосообщение + ссылка на Scorecard
#announcements → новые кейсы, инсайты
#show-your-product → фаундеры постят, мы даём публичный мини-разбор
#work-with-me  → запрос на концепт / аудит
```

Воронка:
```
Соцсети (teardown) → "забери Scorecard" → email → Discord
→ #show-your-product → публичный разбор → DM → концепт → оплата
```
